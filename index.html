1、小数点问题

bug我也说了，首先为什么在末尾输入小数点没有反应，当然不会给你反应，首先就用小数点将字串分割了，当你在末尾输入小数点的时候，分割过后的小数部分的数组为空数组，就认定小数不存在，所以无论你怎么输，都不会显示，解决办法很简单，在输出的时候进行一次判断，当输入的字符串查询到有小数点且小数点位置在末尾的时候，对输出的字串手动加上小数点：

你会发现，此时你在末尾进行操作的时候都可以实现功能，但是输入者并不是只会在末尾进行输入，他可以在内容的任意区域进行输入，这个时候一个巨大的坑就会出现，每输入一个字符，光标就会定位到结束，是不是很无脑，谁会喜欢输入一个字符重新选择一次位置，麻烦得想放弃这个软件，体验十分的差。

回调、同步、异步、阻塞、非阻塞

发表于 2017-09-27 | | 阅读次数

简要介绍

前几天在看es6的时候发现自己对这几个名词的理解并不清楚，发现身边也在学前端的同学也不了解这个，这个毕竟是重要的概念，在前端中也会出现，我再网上进行了相关的查阅，我在这里插入一段对其进行简单的讲解，了解下总有好处，废话不多说，直接进入正题

大家都知道jS本身是单线程运行的，不可能在一段代码还未结束运行时去运行别的代码，因此也就不存在异步执行的概念。而实现异步编程的直接体现就是回调，我先说回调函数。

回调

1、什么是回调函数

举个例子： 相信很多人找过代购买东西，当你要的东西没有的时候，你把你的微信给代购让他有货的时候，就发消息给你，你就支付下单买到你想要的东西。这个例子当做回调的话，你的微信就是回调函数，你把微信给代购就是登记回调函数，当他的东西到了触发了回调关联的事件，代购给你发消息叫做调用回调函数，你下单成功买到东西就是响应回调事件。

通过上面的例子是不是就对回调函数有了一定的认识，回调函数就是一些库函数需要函数作为参数，在满足情况下在调用它。

###2、 回调并不一定是异步 ###
异步编程依托于回调来实现，但不能说使用了回调后程序就异步化了，我们来看下面的这个例子：

const print = (index, callback) => {
    while (index > 0) {
            index--;
    }
       const count = 1;
    callback(count);
}

print(1000, (count) => {
    console.log(count);
})
console.log("如果是异步我在前面");
//1
//"如果是异步我在前面"


上面的代码虽然使用了回调，但是在print内部会等待代码执行完毕再调用回调，中间不会执行函数外部的打印。

如果某个函数做的事情是创建一个别的线程或进程，并与JS主线程并行地做一些事情，并在事情做完后通知JS主线程，那情况又不一样了。

setTimeout(() => {
    console.log("异步我在后面");
}, 0);
console.log(0);
//0
//""异步我在后面


上面的代码我们就可以看到区别，这次是先执行外部的打印，然后在执行回调函数，因此我们可以认为setTimeout这类JS规范之外的由运行环境提供的特殊函数做的事情是创建一个平行线程后立即返回，让JS主进程可以接着执行后续代码，并在收到平行进程的通知后再执行回调函数。

###3、 回调并不是通知了就马上执行 ###
在写个例子，这些例子在我查看另一个概念的时候发现很多人都写了的，先卖个关子，直接上代码，大家猜一下运行的结果：

function heavyCompute(n) {
    var count = 0,
        i, j;

    for (i = n; i > 0; --i) {
        for (j = n; j > 0; --j) {
            count += 1;
        }
    }
}

var t = new Date();

setTimeout(function() {
    console.log(new Date() - t);
}, 1000);
//忙于下面的函数，在下一个事件循环调用
heavyCompute(50000);

都知道既然让大家猜，肯定不会是1000了，肯定没有这么简单，当然啦，但是为什么呢，一些不注意看setTimeout介绍的人就会出错，认为后面的是死的，到了就必须执行，相信眼尖的各位都看到了代码中我的注释，下一个事件循环调用，那什么是事件循环呢？由于篇幅原因，我将在后面的文章中给大家进行介绍。

##异步、同步、阻塞、非阻塞##
在这里我直接将四个放在一起给大家进行一个讲解，因为对照着看对大家理解有很大的帮助。

同步： 函数方法调用时，不会立即返回值，会等待其中执行完毕返回数据；

阻塞： 与上面不同之处是，等待的时候进程会挂起；

异步： 调用的时候立即返回(非结果)，后面完成时候通过状态回调告知调用者；
非阻塞： 不能立即得到结果时，不会阻塞当前进程。
给大家举个例子，光说知识点太空洞了，例如：借助上面代购的例子，同步就是你在那里等代购给你找有没有你要的东西，一直等到结果，而异步则是代购说先给你找下，如果有结果就打电话给你说；阻塞是你会一直把自己挂起，等到他找还是没找到的结果，而非阻塞则是不管代购有没有结果，你自己先做点事情，然后偶尔过来看下结果；

调皮的this

发表于 2017-08-16 | | 阅读次数

简要介绍

每当我们用到函数对象的时候，总会使用this(this在面向对象编程中非常重要),初学者总是搞混this到底代表的是什么，这总会导致得到和自己想的大相径庭的结果；

当我们调用一个函数的时候，会暂停当前函数的执行，传递控住权和参数给新函数，除了申明是的形参外，还有传入参数的类数组arguments和this ，总共有四种调用模式：1、方法调用模式；2、函数调用模式；3、构造器调用模式；4、apply和call调用模式；他们在如何初始化this上存在差异，大家把这四个搞清楚就能理解this的指向了；

1、方法调用模式

说到方法调用，就能想到这个函数作为一个对象的属性，当一个对象的方法被调用的时候，this被绑定到该对象，且this到对象的绑定发生在调用的时候，使得函数可以对this高度复用，this可以访问所属对象，并能获取对象的属性和修改属性,提取属性的方法this.name or this[name]；

var person = {
    firstName: "James",
    lastName: "Chen",
    fullName: function() {
        //this绑定到该对象，可以提取对象的属性进行操作
        return this.firstName + " " + this.lastName;
    }
}
var fullName = person.fullName  //james chen

2、函数调用模式

当一个函数并不是一个对象的方法的时候，那么就是被当做一个函数来调用的，此时this被绑定到全局对象window，但是这在语言设计上是一个错误，让内部函数不能帮助外部函数工作，因为内部函数this绑定了错误的值，所以不能共享该方法对对象的访问权。对于这种错误，我们能通过将this赋值给变量来解决；

var person = {
    fisrtName:　"james",
    lastName: "chen"
}
person.sayHello = function() {
    //将this赋值给变量that方便内部函数的调用
    var that = this;  
    var say = function() {
        alert("hello " + that.firstName + " " + that.lastName + "!");
    }
    say();
}
person.sayHello();  //hello james chen!

虽然this绑定到全局对象上，但是我们不需要用传递的参数this而将需要的对象this赋值给变量that，内部函数通过作用域链就能找到外部函数的变量that,在全局定义的函数this都绑定的是全局变量，注意：在计时器中的this被绑定到了全局变量，在使用的时候需要小心；

3、构造器调用模式

构造器我介绍创建对象的时候已经讲过一次了，这里只简单回顾一下。JavaScript是一门原型继承的语言，当今大多数语言都是基于类的语言，所以它提供了一套和基于类的语言类似的对象构建的语法。通过在函数前面加一个new调用它，这样会创建一个连接到该函数prototype成员的新对象，this就会被绑定到该对象上；new会改变return的行为，new表达式返回值即使return值。否则返回的是构造函数的函数体形成的对象。

function Person() {
    this.firstName = "james";
    this.lastName = "chen"
};
Person.prototype.sayHello = function(){
    alert("hello " + this.firstName + " " + this.lastName);
}
var myPerson = new Person();

4、apply和call调用模式

在JavaScript中函数也可以拥有方法，apply和call基本相似。apply方法让我们传递两个参数，第一个就是我们期望的this的指向，第二个就是存放内部需要使用参数的数组。这样调用后，this就会被绑定到我们指定的对象上；

var num = [3, 4]
function add(x, y) {
    return (x + y);
}
var sum = add.apply(null, num);  //7

call方法和apply差别之处就是call传递的第一个参数后面的参数不是数组，而是用’,’隔开传递给内部使用的参数。

apply方法很有用，我们可以用来处理一些值，如arguments这样的类数组，让它和数组可以进行操作，这里我们将使用Array.prototype.slice()方法；

var arr1 = [1, 2, 3];
function arr2() {
    var arr2 = Array.prototype.slice.apply(arguments);
    return arr2.contact(arr1);
}
arr2([4, 5, 6]);  //[1, 2, 3, 4, 5, 6]

详谈JavaScript中的闭包

发表于 2017-08-13 | | 阅读次数

闭包的定义

有不少的学习者总是将闭包和自执行函数搞混，认为匿名函数就是闭包；其实匿名函数是一个概念：有权访问另一个函数作用域中的变量的函数；

1、回顾

前面我介绍了垃圾回收和作用域的概念，不清楚的可以看下我前面的文章，当一个函数执行完毕其当前的函数作用域就会释放，为了后面可以访问该函数作用域中的变量，我们将使用闭包，先介绍一个例子：

function sum () {
    var a = 10,
        b = 20;
    return function (c) {
        return a + b + c;
    }
}
var add = sum();
document.writeln(add(1));    //31

在这个例子中，在内部函数（匿名函数）中，访问了外部函数中的变量a、b,当这个函数返回的时候，在其他地方被调用，仍然可以访问到这两个变量，因为，内部函数的作用域链包含sum()的作用域；由此可见对于作用域链的了解对闭包的学习至关重要，当某个函数被调用的时候，会创建一个执行环境和相应的作用域链，用arguments和其他命名参数的值来初始化函数的活动对象，在作用域链中，外部函数的活动对象始终处于第二位，外部函数的外部函数的活动对象处于第三位，这样一直到作为终点的全局执行环境；

2、使用闭包

在很多才开始学习的人对闭包并不能理解，为什么要访问其他函数作用域中的东西，我将要用到的东西全部放在全局作用域不就可以了？这就涉及了全局变量特性的弊端了；

2.1、全局变量(一个糟糕的依赖)

全局变量就是在所有作用域中都可见的变量。在开始的时候会带来方便，但是当程序变得越来越复杂的时候，它们将变得难以管理，因为它们在程序的任何部分的任何时间修改，降低了程序的可靠性，全局变量使得同一个程序中的独立子程序运行出错，因为全局变量中的变量恰好等于子程序中的变量时，就会相互冲突；

2.2、拥有自己的私有变量

function hello (name) {
    return function () {
        alert("hello "+ name + "!");
    }
}
var sayhello = hello("Jack");
sayhello();    //hello Jack!

只有通过sayhello()访问到hello函数中的name变量，因为hello()执行完毕后其执行环境的作用域链会被销毁，但它的活动对象仍然会留在内存中，因为匿名函数的作用域链仍然在引用这个活动对象，私有这个概念对创建保护隐私的对象中很有用；

2.3、糟糕的例子

这里我们需要理解的是，内部函数能访问到外部函数的实际变量而无须复制是很重要的；

var addEvents = function (targets) {
    var i,
        len;
    for (i = 0, len = targets.length; i < len; i +=1) {
        targets[i].onclick = function (e) {
            alert(i);    //总会弹出总节点的数目
        }    
    }
};

上面结果出现时因为：当进入循环的时候，i递增且按顺序给节点绑定点击事件和处理函数，此时并不会马上执行事件处理函数，当全部节点都绑定了事件的时候，i的值已经增加为节点的数目，此时点击事件触发的时候，处理函数在它的内部找不到变量i就会沿着作用域链向上查找到addEvent中的变量i，但是i已经递增到了节点数量。
解决方法：直接调用函数，或则将i保存在其他地方，改进(避免在循环中创建函数)后：

var addEvents = function (targets) {
    var i,
        len;
    var handlers = function (i) {
        return function (e) {
            alert(i);
        };    
    };
    for (i = 0, len = targets.length; i < len; i += 1) {
        target[i].onclick = handlers (i);
    }

}
var addEvents = function (targets) {
    var i,
        len;
    for (i = 0, len = targets.length; i < len; i += 1)    {
        (function (i) {
            target[i].onclick = function (e) {
                alert (i);
            }
        })(i)
    }
}

闭包是个很强大的功能，这里只是对他进行一个简单的介绍，大家需要记住闭包就是有权访问另一个函数作用域中的变量的函数，我们可以用闭包做出很多的功能，更多的知识后面我会补充给大家。

javascript的作用域

为了后面介绍的闭包，我先在这里介绍JavaScript中的作用域，作用域控制着变量与参数的可见性和生命周期。

JavaScript的一个重要的概念就是执行环境，每个执行环境都有一个与之相关的变量对象，环境中定义的所有变量和函数都保存在这个对象中；其中最外围的执行环境是全局执行环境，在web浏览器中，全局执行环境被认为是window对象，所有全局变量和函都作为window对象的属性和方法创建的，所以全局中的函数内部的this指向的是window（万事不是绝对的，也可以通过call()和aplly()方法改变this的指向，这里就不做详细的介绍，知道全局中的函数this可以改变指向就可以了，默认情况指向window）

function test(){
    alert(this);    
}
test();   //window对象

每个函数都有自己的执行环境，当执行某个函数的时候，函数的环境会推入一个环境栈中，函数执行完毕后，栈将其环境弹出，把控制权返回给之前的执行环境。

当代码进入一个环境中执行时，会创建变量对象的一个作用域链，其作用是保证对执行环境有权访问的所有变量和函数的有序访问。作用域的前端是当前正在执行代码所在的环境变量对象；其工作机制是向上查找机制，意思是在当前代码执行环境中找不到要求的变量和函数就会向其包含(外部)环境中查找，如果还是没有就向外部的外部环境中查找，以此类推一直找到window对象，只要找到就停止向上查找，注意不能反向查找；

//向上查找机制
var age = 30;
var sex = "男";
function person(){
    name = "小明"；
    var age = 18;
    console.log(age); 
    console.log(sex);
}
person();  //18  男
console.log(name); //小明

上面的代码可以看出，当执行person的时候，执行环境时函数，在内部查找有age变量，所以停止向上查找打印全局中的age变量的值，而sex在函数环境中没有，根据作用域向上查找到window对象中的sex变量的值并打印，相信都注意到最后一行的代码了，这里我是在全局中打印name变量，按理来说我并没有在全局中定义这个变量，为什么不是undefined呢？仔细看person函数内部定义了变量name但是没有使用关键字var，这样定义的变量会直接定义在全局中，使全局也可以使用这个变量，但是不建议大家使用，过度使用会造成污染全局变量，并且在严格模式下这样会导致错误；

在JavaScript中没有块级作用域，只有函数作用域，在其他类的C语言中，由花括号封闭的代码都有自己的作用域，所以在JavaScript中if和for语句定义的变量在其运行完毕后仍然存在，他们定义的变量添加到了当前的执行环境中，而在函数内部定义的变量，在函数内部的任何地方都可见；

//没有块级作用域
for (var i = 0; i < 10; i++){
    //运行相应的代码
}
alert(i);  //10

注意：由于JavaScript没有块级作用域，所以推荐在函数体的顶部定义所有的变量。

JavaScript的垃圾回收

发表于 2017-06-30 | | 阅读次数

js中的垃圾回收机制

和其他的C、C++之类的语言不同，js具有自动垃圾收集机制，其原理大致是：按照固定的时间间隔周期性的找出那些不需要再使用的的变量，将它所占用的内存释放。例如局部变量，一旦函数结束，局部变量就没有存在必要了，可以释放它们占用的内存。貌似很简单的工作，为什么会有很大开销呢？这仅仅是垃圾回收的冰山一角，就像闭包，貌似函数结束了，其实还没有，垃圾回收器必须知道哪个变量有用，哪个变量没用，对于不再有用的变量打上标记，以备将来回收。用于标记无用的策略有很多，常见的有两种方式

标记清除

在js中最常用的垃圾收集方式就是标记清除，其工作方式是：当一个变量进入环境时，将其标记为进入环境（可以使用很多方式来进行标记，例如翻转某个特殊位来记录，或者使用一个“进入环境”的变量列表和“离开环境”的变量列表来跟踪变量的变化），如何标记并不重要，关键是采用什么策略；

垃圾回收器会在运行的时候给存储在内存中的所有变量加上标记，然后去掉环境中的变量以及被环境中变量所引用的变量，在这些完成之后仍存在标记的就是要删除的变量了，因为环境中的变量已经无法访问到这些变量了，然后垃圾回收器相会这些带有标记的变量机器所占空间；低版本的ie采用第二种垃圾收集策略；

引用计数

引用计数的含义就是跟踪记录每个值被引用的次数，当声明了一个变量并将一个引用类型赋值给该变量的时候这个值的引用次数就加1，如果该变量的值变成了另外一个，则这个值得引用次数减1，当这个值的引用次数变为0的时候，说明没有变量在使用，这个值没法被访问了，因此可以将其占用的空间回收，这样垃圾回收器会在运行的时候清理掉引用次数为0的值占用的空间。

很快就遇到了很严重的问题循环引用：对象A中包含一个指向对象B的指针，而对象B也包含一个指向对象A的指针，如下所示：

function test(){
    var objA = new object();
    var objB = new object();
    objA.someObject = objB;
    objB.anotherObject = objA;
}

上面的两个对象通过各自的属性相互引用（次数都是2）；在标记清除的策略中没问题，在引用计数时，函数执行完毕后，两个对象仍然继续存在，因为他们的次数永远不为0；大家都知道，IE中有一部分对象并不是原生js对象，COM对象（BOM和DOM）就是采用引用计数，所以在IE中涉及到COM对象就存在循环引用问题，如下：

var    element = document.getElementById("element");
var obj = {};
element.obj1 = obj;
obj.element = elememt;

既然出现了问题就会有相应的解决方法，最好是在不使用它们的时候手动断开原生js对象与DOM对象之间的连接，如下：

element.obj1 = null;
obj.element = null;

对象的简介

JavaScript是一门面向对象的语言，面向对象的标志是有类的概念，通过类可以创建任意多个具有相同属性和方法的对象。ECMA-262将对象定义为“无序属性的集合，其属性可以包含基本值、对象或者函数。”我的理解是对象就是一些属性和方法组合在一起而构成的一个数据实体。

对象的创建

1、创建一个Object实例

//创建实例，返回的引用保存在一个变量中，相当于c中的指针，指向创建的实例
  var person = new Object();
//添加实例和方法
  person.name = "Rose";
  person.age = 19;
  person.sayhello = function(){
      alert("hello" + this.name);
  } 
//使用对象点属性名  
  alert(person.name)        //Rose

2、使用对象字面量

//将所有属性和方法放在方括号内，使用逗号分开，属性名的引号可有可无
 var person = {
    name: "Rose",
    age: 19,
    sayhello: function(){
      alert("hello" + this.name);
    }
 };

3、构造函数模式

还有工厂模式（用函数来封装以特定借口创建对象的细节），但由于没有解决对象识别的问题，不建议使用，故此处不做详细介绍，提出的解决对象识别的方法是构造函数模式；

function Person(name, age){
    this.name = name;
    this.age = age;
    this.sayhello = function(){
        alert(this);
    }
}
//使用new调用构造函数返回一个Person实例
var person = new Person("Rose", 19);

代码中可以看出，与普通的函数没多大区别，其中的this为调用构造函数的对象（person），构造函数模式创建的对象拥有自己的原生构造函数，实例的cunstructor属性返回构造函数，obj instanceof object 判断是否是某个的实例

4、原型的引入

上面的介绍的构造函数有一定的缺陷，就是每个方法在每个实例上重新创建一次，这样就会降低性能。我们所创建的函数都有一个prototype属性，该属性指向一个对象,包含所有特定类型实例共享的属性和方法。 

function Person(){}
 Person.prototype.name = 19;
 Person.prototype.sayhellp = function(){
    alert(this.name);    
};

注意必须是构造函数创建的对象才能使用prototype属性处理原型，如用对象字面量就不能使用；注意：如果直接给Person.prototype是
重写原型操作，将切断现有原型与任何之前已经存在的对象实例之间的联系，它们引用的仍然是最初的原型。简而言之就是，在创建一个实例后再重写原型，这个实例仍然指向重写前的原型；
最好将原型模式和构造函数模式结合起来使用，使创建的每个实例既有自己独有的属性，又有公有属性；

继承

原型链作为实现继承的主要方法，利用原型链让一个引用类型继承另一个引用类型的属性和方法，实现的本质是重写原型链，将一个实例赋给一个构造函数的原型；下面是组合继承：

function Super(name){
    this.name = name;
}
Super.prototype.sayname = function(){
    alert(this.name);
}
function Sub(name， age){
//在Sub环境中调用Super构造函数，并有自己的属性age
    Super.call(this, name);
    this.age = age;
}
//将Suoer实例赋给Sub的原型
Sub.prototype = new Super();
//增强原型，重写后的原型需要重新指向构造函数
Sub.prototype.constructor = Sub;

总结：上面只是对如何简单的创建对象进行了一个介绍，在实际工作中，每个方法都有用，要根据你实际的情况选择，或许你会觉得将原型没有什么用。
